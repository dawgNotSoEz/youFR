Step 2 Audit Report — Fusion Engine (Multi-Verifier Decision Layer)
Date: 2026-03-18
Target file: services/aggregator/aggregate.py
Pipeline integration: pipeline/main_pipeline.py

Fusion logic implemented
- Added function: fuse_results(llm_result, evidence_result)
- Input format per claim is supported as:
  {
    "claim": str,
    "llm": {"status": str, "confidence": float},
    "evidence": {"status": str, "confidence": float}
  }
- Strict decision logic implemented exactly:
  1) If either verifier is FALSE -> final_status FALSE, reason "At least one verifier marked FALSE"
  2) Else if statuses disagree -> final_status UNCERTAIN, reason "Disagreement between verifiers"
  3) Else if min confidence < 0.7 -> final_status UNCERTAIN, reason "Low confidence"
  4) Else -> final_status TRUE, reason "Both verifiers agree with high confidence"

Pipeline integration changes
- Replaced old hallucination decision path in main pipeline with aggregation-based fusion output.
- For each claim, pipeline now builds fusion input using:
  - llm status/confidence
  - evidence status/confidence
- Added mandatory per-claim debug output in pipeline:
  - claim
  - llm status
  - evidence status
  - fused result
- Final claim decision source is fused output from aggregator.

Aggregate metrics implemented
- total_claims
- false_count
- uncertain_count
- uncertain_ratio

Final decision rule implemented
- hallucination = True when:
  - any fused claim is FALSE
  OR
  - uncertain_ratio > 0.5
