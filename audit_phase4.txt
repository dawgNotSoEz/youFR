1) Summary of all changes

Task 1 — Improved Wikipedia retriever
- Added new live retriever module: services/retriever/wiki_retriever.py
- Implemented query rewriting (examples: Einstein -> Albert Einstein; relativity -> +1905 +1915 +physics)
- Fetches multiple results and builds top 3 paragraph evidence bundle
- Cleans noise (headers/newlines/extra whitespace)
- Guarantees non-empty evidence with minimum length >= 100 chars via fallback augmentation
- Returns structured payload:
  {
    "evidence": <combined text>,
    "sources": <list of titles>
  }

Task 2 — Evidence filter (pre-verification)
- Added: services/verifier/evidence_filter.py
- Implemented is_evidence_relevant(claim, evidence)
  - extracts keywords from claim
  - checks keyword overlap in evidence
  - returns False when overlap is weak
- Integrated in pipeline before evidence verifier call
  - if irrelevant: marks evidence verdict UNCERTAIN and skips verifier API call

Task 3 — Confidence normalization
- Updated services/verifier/groq_verifier.py
- Explicit clamp now applied:
  confidence = max(0.0, min(confidence, 1.0))

Task 4 — Fusion logic upgrade
- Updated services/aggregator/aggregate.py
- Added fuse(llm, evidence) with strict logic:
  - any FALSE -> FALSE ("One verifier FALSE")
  - disagreement -> UNCERTAIN ("Verifier disagreement")
  - low confidence (<0.7) -> UNCERTAIN ("Low confidence")
  - otherwise -> TRUE ("Agreement with high confidence")
- Kept fuse_results wrapper for compatibility

Task 5 — Hard test cases
- Added tests/hard_cases.json with the required 5 adversarial claims

Task 6 — Pipeline debug logging
- Updated pipeline/main_pipeline.py

2) Before vs After behavior

Before (observed during earlier Phase 4 run attempt)
- Retrieval could return empty/weak evidence for some claims
- Evidence branch sometimes became UNCERTAIN due to missing relevance support
- Fusion often became UNCERTAIN because V1 and V2 disagreed

After
- Evidence is structured, cleaned, and length-hardened (>=100 chars)
- Relevance filter blocks weak evidence from reaching verifier
- Fusion uses stricter, deterministic reasons and confidence checks
- Main relativity query now reaches TRUE/TRUE agreement and fused TRUE per claim
